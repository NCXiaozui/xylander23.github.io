Docker学习笔记

Posted at 2019-11-11 CS  Docker 

[TOC]
Docker是一个利用容器(container)来帮助开发人员或系统管理员建立、分享和运行应用的平台。使用容器去部署应用的方式叫做containerization。

Docker 基本概念

Container容器

容器主要是用来提供应用一个隔离的运行环境。

images镜像

镜像提供应用需要的一切东西：代码、运行时间和信赖等等。

分层存储

Docker采用了UnionFS技术，将镜像设计成分层存储的架构。
镜像构建时，会一层层构建，前一层是后一层的基础。每一层构建完就不会再发生改变，后一层的任何改变只发生在自己这一层。比如，删除前一层的文件，其实不是真的删除，而是在当前层标记为已删除，然后最终容器是看不到这个文件的。所以在每一层构建的时候，应该尽量精简。

Containers容器与Virtual machines虚拟机

容器与虚拟机的概念，作用看起来挺类似的，都是为了给应用提供一个与主机隔离的环境。但是区别在于它们的实现方式不同，容器是与主机共享主机的内核(kernel)，但是虚拟机是需要运行一个完整的系统在主机上面。因此，容器其实会更方便，更轻捷。

Dockerfile

Dockerfile是构建镜像的必要文件，通过对Dockerfile的编译，可以构建出镜像，从而运行相应的容器。

常用指令说明

FROM:
FROM <image>[:<tag>] [AS <name>]
FROM指令是为了帮助初始化一个新的构建阶段，同时设置为后续的指令提供基本镜像。FROM可以在一个Dockerfile出现多次去建立镜像或者作为另一个的信赖。
RUN:
RUN <command> | RUN ["executable", "param1", "param2"]
RUN指令会在新的层上面执行，并提交结果，生成的结果也为被下一层使用。（用一次RUN就会建立新的一层）

CMD

CMD指令只能有一个，如果有多个的话，那么只会以最新一个为准。

EXPOSE
EXPOSE <port> [<port>/<protocol>...]
EXPOSE告诉Docker需要在运行的时候监听哪个端口。你同时也可以指定是TCP或是UDP，默认是TCP。
但是要说明一点的是，EXPOSE并没有真正的打开这个端口，真正要打开这个端口是需要在docker run -p指定。

ENV

ENV设置环境变量，后续指令都可以使用。把这个想像成函数的参数，这样就可以只通过使用不同的环境变量，制造出更多的镜像。

ARG

ARG与ENV类似，但是在将来容器运行时不会存在。但是也不要用ARG保存密码等隐私信息，因为docker history是可以看到的。
COPY

COPY将源路径的文件或者字典加到指定的文件系统里面去。
源路径支持使用通配符的写法，目标路径可以写成绝对路径或者是基于WORKDIR的相对路径。
所有新的文件或者目录都会被0号UID和GID创建，除非加了参数--chown指明。（但是如果root filesystem没有包含/etc/passwd或者/etc/group，而且用户和组名已经被使用了，那么就会失败）

ADD

ADD可以理解为高级的COPY，因为它还支持URL和镜像。说明一点，如果加的是一个压缩文件，比如gzip,bzip2以及xz的情况下，ADD会自动解压到目标路径上面去的。
通常还是推荐大家使用COPY而不是ADD，因为COPY语意更加清晰明了。

WORKDIR
WORKDIR /path/to/workdir
WORKDIR主要是为了后面RUN、CMD等提供一个工作目录。如果工作目录不存在，会被Docker创建。

操作容器

启动容器

启动容器有两种方式，一种是根据新建镜像新建一个容器并启动，另一个是终止状态下重新启动。
新建启动
docker run 之前教程上面，在Dockerfile目录下面，执行一个docker container run就会根据新的镜像建立一个新的容器并启动它。
启动已终止容器
docker container start命令，可以将一个已经终止的容器启动运行。

终止容器

docker container stop来终止运行容器

进入容器

有两种方式可以进入容器，一种是attach，一种是docker exec
attach
docker attach container_id
exec
exec -i -t contain_id bash
这里的-i -t一起使用可以出现Linux命令提示符。

删除

docker container rm container_id
