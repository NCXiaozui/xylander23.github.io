Docker学习笔记

Posted at 2019-11-11 CS  Docker 

[TOC]
Docker是一个利用容器(container)来帮助开发人员或系统管理员建立、分享和运行应用的平台。使用容器去部署应用的方式叫做containerization。

Docker 基本概念

Container容器

容器主要是用来提供应用一个隔离的运行环境。

images镜像

镜像提供应用需要的一切东西：代码、运行时间和信赖等等。

Containers容器与Virtual machines虚拟机

容器与虚拟机的概念，作用看起来挺类似的，都是为了给应用提供一个与主机隔离的环境。但是区别在于它们的实现方式不同，容器是与主机共享主机的内核(kernel)，但是虚拟机是需要运行一个完整的系统在主机上面。因此，容器其实会更方便，更轻捷。

在Ubuntu安装Docker

安装Docker引擎

这里使用的是用repository的方式安装

  1. 更新package索引
    sudo apt-get update

  2. 安装packages让apt可以通过HTTPS的方式使用repository

  3. 增加Docker’s 的官方密钥
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
    这里可以验证一下密钥是不是正确的：

  4. 设置稳定版的repository

  5. 再次更新一下package索引，然后安装Docker引擎

  6. 尝试运行Docker，确认是否安装完成
    sudo docker version
    因为Docker需要用户具有sudo权限，所以为了避免输入sudo，你可以把用户加入Docker用户组：
    sudo usermod -aG docker $USER

Containerizing 一个应用

1. 克隆一个样例

2. 构建Dockerfile文件

Dockerfiles描述如何去实现一个私有容器的文件系统，而且也说明基于这个镜像需要包含哪些元素去运行容器。
样例：

  • FROM 导入一个已存在的镜像。在这个文件里面是node:6.11.5，这是一个官方镜像。
  • WORKDIR 在镜像文件系统里面指定从这个目录下面执行后面的指令。这里是/usr/src/app
  • COPY 把package.json(/usr/src/app/package.json)从主机复制到镜像文件里面的位置(.)
  • RUN 在镜像文件系统里面执行npm install
  • CMD 告诉镜像文件系统有哪些操作行为。

3.构建镜像

sudo docker image build -t bulletinboard:1.0 .i

4.启动镜像

sudo docker container run --publish 8000:8080 --detach --name bb bulletinboard:1.0

  • publish 指定主机端口入口是8000，容器的端口是8080
  • detach 后台运行
  • name 给容器取个名字
    然后现在你可以访问localhost:8000,应该可以看到下面这个页面：

    删除镜像

    docker container rm --force bb
